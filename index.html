Multiple Inputs: When Life Gets More Complex

Of course, real life is rarely as simple as predicting house prices based on just size. What about the number of bedrooms, the age of the house, or the distance to the nearest coffee shop (a crucial factor in some neighborhoods!)?

Let's see how both approaches handle multiple inputs. In linear regression, we simply add more terms:

\[ \text{Price} = m_1(\text{size}) + m_2(\text{bedrooms}) + m_3(\text{age}) + b \]

Continue

Our neural network does exactly the same thing, just with its own terminology:

\[ \text{Price} = w_1(\text{size}) + w_2(\text{bedrooms}) + w_3(\text{age}) + b \]

Build Your Vocab

Weights

The neural network equivalent of regression coefficients. They determine how much importance the network gives to each input.

Continue

Mathematicians love to write this more compactly using vector notation (and who can blame them?):

\[ y = \mathbf{w}^T\mathbf{x} + b \]

Stop and Think

Why do you think using vector notation is helpful?

Imagine writing out an equation with 1000 inputs! Vector notation keeps things clean and makes it easier to work with large-scale problems.
